# AI Attribution

This document discloses the use of artificial intelligence tools in the development of this project, in accordance with academic integrity requirements for the Valley City State University M.S. in Business Information Systems capstone.

## AI Tool Used

- **Tool:** Claude (Anthropic)
- **Model:** Claude Opus 4.6
- **Interface:** Claude.ai (web/mobile chat with computer use)
- **Period of Use:** March 2026

## How AI Was Used

### Code Development
Much of the python application code, HTML templates, CSS styling, and JavaScript was developed collaboratively through iterative conversation with Claude. The development process involved:

- Describing requirements and use cases in natural language
- Receiving generated code from Claude
- Testing on live infrastructure and reporting results back to Claude
- Iterating on fixes based on real API responses (e.g., discovering `keep_unlock` vs `keep_open`, correct API field names, correct API base paths)
- Requesting feature additions (camera groups, exclusion lists, portal tabs) through conversation

### What the Human Author Did
- **Architecture decisions:** Chose Flask over heavier frameworks, chose separate services over a monolith, decided on inclusion vs exclusion lists, selected port assignments
- **Infrastructure:** Created the Proxmox VM, installed Ubuntu Server, configured networking/VLANs, deployed and tested all services
- **API discovery:** Tested API endpoints on live UniFi hardware, provided real API responses that drove code corrections
- **Configuration:** Set up systemd services, cron jobs, Gmail app passwords, controller admin accounts, API tokens
- **Validation:** All code was tested against production UniFi Protect (UNVR Pro), UniFi Access, and UniFi Network (Cloud Key Gen2+) infrastructure
- **Iteration direction:** All feature requests, scope decisions, and design choices originated from the human author

## Scope of AI Contribution

| Component | AI Contribution |
|---|---|
| Python application code | Generated by Claude, tested and debugged collaboratively |
| HTML/CSS/JS templates | Generated by Claude |
| Configuration templates | Generated by Claude, populated with real values by human |
| systemd service files | Generated by Claude |
| README files | Generated by Claude |
| Infrastructure setup | Human only (Proxmox, Ubuntu, networking, deployment) |
| Testing and validation | Human only (live hardware testing) |
| Git/GitHub setup | Human with step-by-step guidance from Claude |

## Transparency Note

This project represents a collaborative workflow between a human systems engineer and an AI assistant. The human author provided domain expertise in UniFi infrastructure, cybersecurity, and physical security operations. The AI assistant provided code generation and technical guidance. All code was deployed to and validated against real production infrastructure.
